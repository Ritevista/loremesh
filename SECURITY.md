# Security policy

LoreMesh is pre-release; only the latest `main` branch receives security fixes. Do not open a public issue for a suspected vulnerability. Use GitHub's private vulnerability reporting for this repository. Include impact, affected revision, reproduction steps using non-sensitive fixtures, and a suggested mitigation if available. Maintainers should acknowledge a report within seven days.

Never include real credentials or private source content. LoreMesh performs no telemetry or implicit network access. See `docs/architecture/security-and-privacy.md` for the threat model and disclosure expectations.
